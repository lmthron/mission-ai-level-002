Claude Skills - Fundamentals

Review these key concepts about Claude Skills before taking your Module 1 checkpoint quiz. Understanding Skills is essential for building reusable, specialized AI workflows.

What Are Claude Skills?

The Simple Explanation

Claude Skills are folders of instructions, scripts, and resources that Claude loads dynamically to improve performance on specialized tasks. Think of them as reusable instruction manuals that teach Claude how you want specific tasks done.

Progressive Disclosure - How Skills Load

Claude doesn't load every Skill into memory all the time. Instead:

  • Claude keeps only lightweight metadata (name and description) for each Skill
  • When your request matches a Skill, Claude loads just that Skill's full instructions
  • This keeps Claude fast while giving it access to specialized expertise

Skills Are Portable

A Skill you create works in Claude web, Claude Code (terminal), and the Claude API without modification. Build once, use everywhere.

Basic Skill Anatomy

SKILL.md File Structure

---
name: your-skill-name
description: Clear description of when to use this skill

# Your Skill Name

[Instructions for Claude go here]

## Examples

- Example 2

- Guideline 1
- Guideline 2

Critical Components

  • YAML Frontmatter (REQUIRED): Name (lowercase-with-hyphens only) and description
  • Instructions (REQUIRED): Clear, step-by-step guidance for Claude
  • Examples (Highly Recommended): Show Claude what success looks like
  • Guidelines (Optional): Constraints or requirements to follow

‚ö†Ô∏è YAML Name Field Requirements

Must use: Lowercase letters, numbers, and hyphens only (64 characters max)

Good: brand-guidelines, meeting-notes-formatter, data-analyzer-v2

Bad: Brand Guidelines, Meeting_Notes, Data Analyzer

Note: The description field CAN use capitals, spaces, and punctuation.

Creating Skills - Two Methods

Method 1: Using skill-creator (If Available)

  1. Enable skill-creator in Settings > Capabilities > Skills
  2. Say: "I want to create a skill for [what you need]"
  3. Answer Claude's questions about the task
  4. Provide any reference materials
  5. Download the ZIP file Claude creates
  6. Upload the ZIP to Settings > Capabilities > Skills

Method 2: Manual Creation (More Control)

  1. Create a folder for your Skill
  2. Create SKILL.md file inside the folder
  3. Add YAML frontmatter and instructions
  4. (Optional) Add resource files (logos, templates, examples)
  5. Compress the entire folder to ZIP format
  6. Upload ZIP to Settings > Capabilities > Skills

‚ö†Ô∏è CRITICAL: Enable Code Execution First

Skills require Code Execution to be enabled in Settings > Capabilities. Do this BEFORE creating or uploading Skills, or they won't work. If your Skill hangs or doesn't respond, Code Execution is usually why.

Adding Resources to Skills (Optional)

What You Can Include

  • Logos (.png, .svg, .jpg): Claude inserts into documents
  • Templates (.pptx, .docx, .xlsx): Claude uses as starting points
  • Data examples (.csv, .json, .txt): Reference formats
  • Style guides (.pdf, images): Visual brand standards

‚ö†Ô∏è Important Size Limit

Claude has an 8MB uncompressed size limit for skill uploads. Large template files often exceed this limit. Solution: Create separate skills (one for logos/colors, another for templates) or use streamlined template files.

How to Invoke Skills

Method 1: Mention the Task Naturally

Just describe what you need. If it matches a Skill's description, Claude will use it automatically.

Example: "Create a quarterly business review presentation"

Method 2: Reference by Name

Explicitly mention the Skill name in your request.

Example: "Use the Meeting Notes Formatter to structure these notes"

Method 3: Upload Related Materials

Upload a document that triggers the Skill context automatically.

Testing Your Skills

Three Tests to Run

  1. Recognition Test: Does Claude recognize and load your Skill? Check the thinking process.
  2. Output Test: Does it produce what you specified? Check format, structure, guidelines.
  3. Edge Cases Test: Try with minimal information, lots of details, missing elements.

Refine If Needed

  • Download your Skill ZIP and extract it
  • Edit SKILL.md to be more specific
  • Add more examples if output wasn't right
  • Re-zip and re-upload
Skill Management Best Practices

Organization

  • Keep Skills focused on one task or workflow
  • Don't create mega-Skills that try to do everything
  • Name Skills clearly and descriptively

Maintenance

  • Update Skills when your processes change
  • Remove Skills you no longer use (20 Skill limit on web)
  • Version your Skills if you're iterating

Testing

  • Always test new Skills before relying on them
  • Try edge cases and unusual inputs
  • Verify output quality meets your standards

Security

  • Only install Skills from trusted sources
  • Review any Skill before enabling it
  • Don't include sensitive information in Skill files (passwords, API keys)
