Structured Outputs and Data Formats

Quick Reference Guide

This guide summarizes the key concepts from Module 3: how to get Claude to produce structured data, when to use different formats, and how to connect AI-generated data to your workflows.

What is Structured Data?

The Core Concept

Structured data is information organized in a predictable, consistent format that computers can easily parse and process.

The Key Difference: Human-readable reports use paragraphs and formatting. Machine-readable data uses consistent, parseable formats.

Why This Matters

Without structured outputs, AI is just a writing tool.

With structured outputs, AI becomes a data generation system that connects to your actual workflows.

  • Import into Excel or databases
  • Sort, filter, and analyze
  • Feed into other systems automatically
  • Validate for completeness and accuracy
The Three Main Formats

üìÑ CSV

Simple text format where commas separate fields. Each line is a record.

Best for: Excel imports, simple tabular data, data analysis Avoid for: Nested structures, data with commas, API integrations

üîß JSON

Key-value pairs with support for nested structures. Standard for modern systems.

Best for: APIs, configuration files, complex nested data Avoid for: Non-technical users editing manually, simple flat data for Excel

üìã XML

Markup language with tags. Less common in modern workflows.

Best for: Legacy systems, SOAP APIs, specific system requirements Avoid for: New projects (use JSON), simple data (use CSV)
Quick Decision Rule:

Going into Excel? Use CSV. Going into an API or code? Use JSON. System specifically requires XML? Use XML (but confirm it's truly needed).

Getting Claude to Produce Structured Data

The Three Essential Steps

  • Step 1: Specify the format explicitly (CSV, JSON, XML)
  • Step 2: Define the structure clearly (column names, data types, formatting rules)
  • Step 3: Give examples when structure is complex

‚ùå Bad Prompt

Generate some customer data

‚úì Good Prompt

Generate 10 sample customer records in CSV format with these columns: customer_id, customer_name, email, company, signup_date

üí° Pro Tips for Better Prompts

  • Specify number formats: "2 decimal places" or "whole numbers only"
  • Define date formats: "YYYY-MM-DD" is standard and sorts correctly
  • Clarify data types: "numbers not in quotes" for JSON
  • Request realistic data: "realistic but fake names and emails"
  • Ask for downloadable files: "Create as a downloadable CSV file"
Validation: Catching Errors Before Import

Why Validate CSV

Validation ensures your CSV structure is consistent so systems can import it correctly. It checks if every row has the same number of fields, if values with commas are properly escaped, and if the date format is consistent.

What validation does NOT check: Whether your data is accurate or meets business rules.

CSV Validation Methods

  • Method 1: Import into Excel and verify columns align properly
  • Method 2: Upload to Claude and ask it to check for errors
  • Method 3: Open in text editor and scan for inconsistent commas

Why Validate JSON

Validation checks if your JSON syntax is correct so systems can parse it. It verifies all brackets are matched, commas are in the right places, and strings are properly quoted.

What validation does NOT check: Whether values make business sense or field names match expectations.

JSON Validation Methods

  • Method 1: Use JSONLint.com to check syntax
  • Method 2: Upload to Claude and ask it to validate structure
  • Method 3: Use Python's json.load() if you code
Common Errors and Fixes

CSV Errors

‚ùå Error: Commas in data break structure

Problem: Joe's Coffee, Bakery,15.50 creates extra column

‚úì Fix: Wrap in quotes "Joe's Coffee, Bakery",15.50
‚ùå Error: Inconsistent number of columns

Problem: Some rows missing fields

‚úì Fix: Include empty values, ensure every row has same field count
‚ùå Error: Date format inconsistency

Problem: Mix of 01/15/2024, 2024-01-16, Jan 17, 2024

‚úì Fix: Use consistent format, prefer YYYY-MM-DD

JSON Errors

‚ùå Error: Missing comma between objects

Problem: {"id": 1} {"id": 2}

‚úì Fix: Add comma {"id": 1}, {"id": 2}
‚ùå Error: Trailing comma after last item

Problem: {"name": "Alice", "email": "alice@example.com",}

‚úì Fix: Remove trailing comma
‚ùå Error: Numbers as strings

Problem: {"amount": "150.00", "quantity": "5"}

‚úì Fix: Remove quotes from numbers {"amount": 150.00, "quantity": 5}
The 5-Step Workflow Pattern

Connecting AI Output to Business Processes

Step 1
Define Requirements
What format does the destination system need? What fields are required?
Step 2
Generate with Claude
Get structured data in the required format with proper specifications
Step 3

Critical Point:

You must know the target system's requirements BEFORE generating data. Don't generate first and hope it works.

Format Decision Framework

Start Here: What's the Destination?

  • Going into Excel/Google Sheets? ‚Üí Use CSV
  • Going into a database or API? ‚Üí Use JSON
  • Legacy system specifically requires XML? ‚Üí Use XML (confirm first)
  • Non-technical users need to edit it? ‚Üí Use CSV
  • Data has nested structures? ‚Üí Use JSON
  • Simple flat data (no nesting)? ‚Üí Use CSV

‚úÖ When to Use CSV

  • Importing into Excel or Google Sheets
  • Simple tabular data (rows and columns)
  • Data analysis and reporting
  • Non-technical people need to view/edit
  • Flat data structures (no nested relationships)

‚úÖ When to Use JSON

  • API requests and responses
  • Configuration files for applications
  • Complex nested data structures
  • Data consumed by code/systems
  • Need to preserve data types (numbers, true/false)

‚ö†Ô∏è About XML

XML is less common in modern workflows. Only use it if:

  • A system specifically requires XML format
  • Working with legacy enterprise software
  • Dealing with SOAP APIs (older standard)

If you don't know if you need XML, you probably don't. Focus on CSV and JSON.

Advanced Techniques

Generate Realistic Patterns

Instead of uniform random data, specify realistic distributions:

  • 70% of transactions between $20-$100 (typical purchases)
  • 20% of transactions between $100-$300 (larger purchases)
  • 10% of transactions between $300-$1000 (bulk/premium)

Why this matters: Test data that reflects real patterns catches more issues than uniform random data.

Generate Related Data Across Files

For complex testing, generate multiple related datasets:

  • File 1: Customer records with customer IDs
  • File 2: Orders referencing those customer IDs
  • File 3: Order items referencing the order IDs

This maintains referential integrity across your test data.

Incremental Generation for Large Datasets

For very large datasets, generate in batches:

  • First prompt: "Generate records 1-50, start IDs at TXN0001"
  • Second prompt: "Continue with records 51-100, start at TXN0051"
  • Final prompt: "Combine both batches into single file"
üéØ Key Takeaway:

Structured data transforms AI from a writing tool into a data generation system. Always specify format explicitly (CSV/JSON/XML), define structure clearly (columns, data types, formatting), and validate before importing. Use CSV for Excel, JSON for APIs and configuration, XML only when specifically required. Follow the 5-step workflow: Define Requirements ‚Üí Generate ‚Üí Validate ‚Üí Import ‚Üí Verify.
